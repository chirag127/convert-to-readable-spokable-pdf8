About Spokable PDF

The Problem

Technical documentation, research papers, and educational materials are increasingly consumed through text-to-speech (TTS) systems. Whether you're commuting, exercising, or simply prefer audio learning, TTS apps like Moon+ Reader Pro have become essential tools.

However, standard PDFs contain elements that sound like gibberish when read aloud:

  • Code blocks - Line-by-line reading of syntax is incomprehensible
  • Tables - Raw data without context loses meaning
  • Mathematical equations - Notation like "x^2" or "âˆ‘" isn't spoken naturally
  • Figures and diagrams - Visual information is completely lost

The Solution

Spokable PDF uses Google's Gemini AI to intelligently transform technical content into natural, conversational language optimized for TTS systems.

Instead of reading code line by line, it explains what the code does. Instead of listing table cells, it describes the data relationships. Mathematical notation is converted to spoken words.

How It Works

  1. Extract - PDF.js extracts text, code blocks, tables, and identifies images
  2. Analyze - Content is intelligently chunked and categorized
  3. Transform - Gemini AI converts technical elements into natural language
  4. Reconstruct - A new PDF is generated with TTS-optimized content

Privacy & Security

Your privacy is paramount. Spokable PDF is a 100% client-side application:

  • All processing happens in your browser
  • Your PDF files never leave your device
  • API keys are stored locally in your browser
  • No server-side storage or tracking
  • Open source and auditable

Technology Stack

  • Frontend - Pure HTML5, CSS3, Vanilla JavaScript (ES6+)
  • AI - Google Gemini API (3.0, 2.5, 2.0 series)
  • PDF Processing - PDF.js for extraction, jsPDF for generation
  • Storage - IndexedDB for files, LocalStorage for settings
  • Deployment - Static hosting (GitHub Pages, Vercel)
